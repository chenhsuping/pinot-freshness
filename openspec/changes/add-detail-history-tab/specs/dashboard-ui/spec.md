## ADDED Requirements

### Requirement: 詳情頁籤切換（概覽／歷史）
詳情頁 SHALL 提供頁籤切換控制，含「概覽」與「歷史」兩個頁籤，預設 MUST 為「概覽」。
「概覽」頁籤 MUST 維持既有內容（狀態條「近七天累積 Downtime」、近七天延遲趨勢卡、四列資訊卡），
「歷史」頁籤 MUST 顯示逐筆檢查記錄畫面。切換頁籤 MUST 只重繪當前頁籤內容，不重新抓取資料；
開啟另一張表的詳情時，頁籤 MUST 重設為「概覽」。

#### Scenario: 預設顯示概覽
- **WHEN** 使用者開啟某表詳情頁
- **THEN** 頁籤切換顯示「概覽」為選中態，並呈現近七天累積 Downtime 狀態條、趨勢卡與資訊卡

#### Scenario: 切到歷史頁籤
- **WHEN** 使用者點選「歷史」頁籤
- **THEN** 改顯示歷史記錄畫面（範圍膠囊、摘要列、逐筆記錄表），且不重新抓取資料

#### Scenario: 重開詳情重設頁籤
- **WHEN** 使用者返回後再開啟另一張表的詳情頁
- **THEN** 頁籤重設為「概覽」

### Requirement: 歷史頁籤——範圍篩選與逐筆記錄表
「歷史」頁籤 SHALL 提供日期範圍膠囊（`近24小時` / `近3天` / `近7天`，預設 `近24小時`）、
一列摘要文字，以及逐筆檢查記錄表。範圍切換 MUST 對已載入的近七天歷史於前端切片（24／72／168 小時），
不得重新打 API。記錄 MUST 依 `Check_Time` 由新到舊排序。記錄表 MUST 含四欄：檢查時間、資料更新時間、
延遲時間（人性化、依逾時狀態著色）、逾時狀態（pill「逾時／正常」）；逐筆逾時判定 MUST 為
`Delay_Time > SLA`。摘要列 MUST 為「{範圍標籤} · 共 {N} 筆 · {M} 筆逾時」。

#### Scenario: 預設範圍與摘要
- **WHEN** 使用者首次進入「歷史」頁籤
- **THEN** 範圍為「近24小時」，摘要列顯示「近24小時 · 共 {N} 筆 · {M} 筆逾時」，記錄表列出該窗內逐筆記錄（最新在上）

#### Scenario: 切換範圍前端切片
- **WHEN** 使用者點選「近3天」或「近7天」
- **THEN** 記錄表與摘要依該範圍（72／168 小時）重新切片呈現，且不重新抓取資料

#### Scenario: 逐筆逾時著色與狀態
- **WHEN** 某筆記錄的 `Delay_Time` 大於該表 SLA
- **THEN** 該筆延遲時間以紅色系顯示且狀態 pill 為「逾時」；否則為綠色系且 pill 為「正常」

#### Scenario: 範圍內無記錄
- **WHEN** 所選範圍內沒有任何檢查記錄
- **THEN** 記錄表顯示空狀態（如「此範圍尚無檢查記錄」），摘要列顯示「{範圍標籤} · 共 0 筆 · 0 筆逾時」

### Requirement: 詳情全域頁首來源與 SLA 標示
詳情畫面的全域 sticky 頁首副標 SHALL 在表名與「{BU} · {group}」之外，加上來源 chip 與「SLA {人性化}」。
來源 chip MUST 採用深底頁首變體：Realtime 為 `rgba(77,163,255,.18)` 底 `#8CC4FF` 字、
Offline 為 `rgba(255,255,255,.12)` 底 `#D6DBE4` 字。SLA 文字 MUST 人性化（如 `15分`、`4小時`）。

#### Scenario: Realtime 來源頁首
- **WHEN** 詳情頁的表來源為 Realtime
- **THEN** 頁首副標顯示藍色系來源 chip 與「SLA 15分」

#### Scenario: Offline 來源頁首
- **WHEN** 詳情頁的表來源為 Offline
- **THEN** 頁首副標顯示灰色系來源 chip 與「SLA 4小時」

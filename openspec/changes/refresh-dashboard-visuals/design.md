## Context

先前已依設計稿落地了 sticky 頁首、四個畫面、詳情頁籤/歷史、趨勢 hover、統計磚、卡片微互動與排程空窗警示。最新一版設計稿對「群組卡、詳情頁籤、趨勢點、統計磚標籤、頁首高度」做了精修。本變更把這些視覺差異補進 `dashboard-ui` 規格,純外觀,不動 `core.js` 資料邏輯與導覽 stack。

## Goals / Non-Goals

**Goals:**
- 規格與最新設計稿一致,讓實作有單一依據。
- 沿用既有「Slate Ops」設計 token(僅新增進度條三段色 `#E8A23C`)。

**Non-Goals:**
- 不更動資料模型、查詢、導覽行為、排程偵測邏輯。
- 不重寫模板執行期(附件的 `<sc-for>`/`<sc-if>` 僅為設計稿,讀作規格,以既有原生 HTML 產生器實作)。

## Decisions

- **群組卡加健康進度條**:以視覺長度表達健康率,比純文字更快掃讀。配色三段:全正常 `#34A06B`、有逾時且 ≥50% `#E8A23C`(新增琥珀,僅此用途)、有逾時且 <50% `#E0584A`。
- **頁籤改深藍選中 + 圓點**:提高選中態對比,圓點作為次要視覺指示(選中 `#E8C6CF` 酒紅淡色、未選 `#C9D2DA`)。
- **趨勢每點圓點**:讓 7 天每個取樣點可辨識,並與 hover 空心點區分(每點為實心 `r:2.2` 白描邊;hover 為較大空心點)。
- **統計磚更名「最大值/最小值」**:用語更口語直觀,取代「峰值/谷值」。
- **頁首 `min-height:44px`**:單行首頁與兩行內頁切換時頁首不跳動。

## Risks / Trade-offs

- **既有測試斷言需更新**:`views.test.js` 對「峰值/谷值」的字串斷言要改為「最大值/最小值」;屬預期內小調整。
- **新增一個琥珀色**:僅限健康率中段(逾時但 ≥50%),不擴散其他語意。
- **趨勢點數量多時(168 點/7天)圓點密集**:`r:2.2` 已足夠小,密集時視覺仍可接受;必要時可只在較少取樣的範圍顯示。
